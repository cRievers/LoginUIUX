
### 2. Mapeamento das Heurísticas de Nielsen (Relatório de IHC)

Para o seu relatório, você deve "provar" onde cada heurística foi aplicada no sistema[cite: 1]. Abaixo, apresento o mapeamento inicial sugerido para as funcionalidades pedidas:

| Heurística | Aplicação Prática no Projeto | Tela Relacionada |
| :--- | :--- | :--- |
| **#1 Visibilidade do status** | Indicador visual de qual assento foi selecionado (mudança de cor) e barra de progresso no checkout[cite: 1]. | Mapa de Assentos / Checkout |
| **#2 Correspondência com o mundo real** | Uso de ícones de calendário para datas e um mapa visual que simula a disposição física do auditório[cite: 1]. | Lista de Eventos / Mapa |
| **#3 Controle e liberdade** | Botão "Voltar" ou "Cancelar Seleção" antes de confirmar o pagamento sem perder o fluxo[cite: 1]. | Seleção de Assento / Checkout |
| **#4 Consistência e padrões** | Uso rigoroso de cores (Roxo = Ação, Vermelho = Erro) e botões seguindo o padrão Material Design[cite: 1]. | Geral |
| **#5 Prevenção de erros** | Desabilitar o botão "Comprar" até que um assento e uma categoria de ingresso sejam escolhidos[cite: 1]. | Detalhes do Evento |
| **#6 Reconhecer em vez de relembrar** | Mostrar o resumo do evento (nome, data, lugar escolhido) visível durante todo o processo de pagamento[cite: 1]. | Checkout |
| **#7 Flexibilidade e eficiência** | No perfil, permitir que o usuário veja rapidamente apenas "Próximos Eventos" ou "Eventos Passados"[cite: 1]. | Perfil do Usuário |
| **#8 Estética e minimalismo** | Interface limpa, com espaços em branco (white space) e apenas informações cruciais para a compra[cite: 1]. | Geral |
| **#9 Diagnóstico e recuperação de erros** | Mensagens claras se o cartão de crédito (simulado) for inválido, explicando o porquê em vez de códigos técnicos[cite: 1]. | Checkout |
| **#10 Ajuda e documentação** | Uma pequena seção de "Dúvidas Frequentes" ou dicas flutuantes (tooltips) sobre a diferença entre Normal e VIP[cite: 1]. | Detalhes do Evento |
